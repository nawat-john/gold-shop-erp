# คู่มือการใช้งานระบบ Gold Shop ERP (User Manual)

ระบบ Gold Shop ERP ออกแบบมาเพื่อบริหารจัดการธุรกิจร้านทองแบบครบวงจร ครอบคลุมทั้งหน้าร้าน คลังสินค้า ขายฝาก ออมทอง งานช่าง และการบัญชีแยกประเภท คู่มือนี้แบ่งแยกตามบทบาทผู้ใช้งาน (User Roles) เพื่อความสะดวกในการเรียนรู้และปฏิบัติงาน

---

## 1. เจ้าของร้าน (Owner)

**หน้าที่:** ดูภาพรวมบริหารจัดการ อนุมัติการตั้งค่าและรายการสำคัญของทุกสาขา

- **การเข้าดู Dashboard และรายงานการเงิน:**
  - ไปที่เมนู **การเงิน/บัญชี** (`/admin/accounting`) เพื่อดูรายงานงบทดลอง (Trial Balance), งบกำไรขาดทุน (P&L) และงบแสดงฐานะการเงินเบื้องต้น
  - รายงานแยกประเภทเงินสดและบัญชีธนาคารเพื่อตรวจสอบยอด Running Balance
- **การอนุมัติการทำงานพิเศษ (Maker-Checker):**
  - เมื่อมีคำขอพิเศษ เช่น การอนุมัติทองหลุด, การยกเลิกบิล หรือการปรับสต๊อกระบบจะแสดงกล่องป้อน PIN เพื่อยืนยันตัวตนระดับสูง
- **การตรวจทาน Audit Log:**
  - ตรวจสอบความถูกต้องและประวัติการเข้าใช้งานและทำธุรกรรมของทุกคนผ่านทางเมนู **Audit Log** (`/admin/audit-logs`) ซึ่งไม่สามารถลบหรือแก้ไขได้

---

## 2. ผู้จัดการสาขา (Branch Manager)

**หน้าที่:** ควบคุมดูแลความเรียบร้อยของสาขาตัวเอง อนุมัติราคาทอง และกระทบยอดกะ

- **การประกาศราคาทองคำประจำวัน:**
  - ไปที่เมนู **จัดการราคาทอง** (`/admin/prices`) เพื่อประกาศราคาซื้อ/ขายของทางร้านสำหรับทองแท่งและทองรูปพรรณ (Snapshot ติดไปกับบิล)
- **การปิดกะพนักงานและ Reconcile ยอด:**
  - เมื่อหมดวัน ตรวจสอบและอนุมัติการกระทบยอดลิ้นชักเงินสดผ่านเมนู **กะพนักงาน** (`/admin/pos/shifts`) โดยป้อน PIN อนุมัติเมื่อผู้ใช้งานป้อนยอดเงินปิดกะจริงและพบยอดดิฟ
- **การจัดการความขัดแย้งของสต๊อก (Stock Adjustment):**
  - ในกรณีทองสูญหายหรือชั่งน้ำหนักคลาดเคลื่อนเกินขอบเขต สามารถปรับสต๊อกโดยตรงพร้อมบันทึกหมายเหตุแนบเหตุผล

---

## 3. พนักงานขาย (Cashier/Sales)

**หน้าที่:** ทำธุรกรรมซื้อ-ขายหน้าร้าน, รับจำนำ/ขายฝาก, ออมทอง, รับซ่อมทอง

- **การเปิดกะลิ้นชักเงินสด (Open Shift):**
  - ก่อนทำรายการแรกของวัน ไปที่หน้า **POS** หน้าเปิดกะ เพื่อระบุเลขลิ้นชักเงินสดและใส่ยอดเงินทอนเริ่มต้น (เช่น 5,000 บาท)
- **การขายทองใหม่ (Sales Order):**
  - สแกนบาร์โค้ดป้ายราคาสินค้า ระบบจะดึงน้ำหนักและคำนวณราคาอัตโนมัติตามราคาสมาคมปัจจุบัน + ค่ากำเหน็จ
  - ป้อนข้อมูลชำระเงิน (สามารถแบ่งจ่ายหลายช่องทาง เช่น โอนเงิน 10,000 + จ่ายเงินสดส่วนที่เหลือ)
- **การเปิดสัญญาขายฝาก (Pawn Contract):**
  - ทำการลงทะเบียนหรือค้นหาประวัติลูกค้า (บังคับ KYC บัตรประชาชนหากเกินวงเงิน AMLO)
  - ป้อนรายละเอียดทองคำ, น้ำหนักชั่งจริง, ถ่ายรูปทรัพย์สิน, และป้อนวงเงินสินเชื่อที่อนุมัติพิมพ์สัญญาให้ลูกค้าลงนาม
- **การรับชำระดอกเบี้ยขายฝาก (Renew Interest):**
  - ค้นหาสัญญาของลูกค้าจากระบบ ป้อนจำนวนวันที่ต่ออายุ คีย์ดอกเบี้ยรับสด ปริ้นท์ใบเสร็จดอกเบี้ยให้ลูกค้า

---

## 4. พนักงานสต๊อก (Stock Keeper)

**หน้าที่:** รับทองเข้าคลัง โอนย้ายสินค้าข้ามสาขา และจัดหมวดหมู่สินค้า

- **การรับทองเข้าคลังสินค้า (Stock In):**
  - บันทึกการรับทองจากโรงงานหรือ Supplier พิมพ์ป้ายบาร์โค้ดติดป้ายสินค้า (Serial Number) ระบุน้ำหนักกรัม และค่ากำเหน็จตั้งต้น
- **การโอนสินค้าข้ามสาขา (Branch Transfer):**
  - ทำใบคำขอโอนย้าย เลือกสินค้าที่จะส่งไปสาขาปลายทาง สถานะเปลี่ยนเป็น `IN_TRANSIT` เมื่อสาขาปลายทางได้รับทองและกดยืนยัน ยอดสต๊อกจะย้ายสาขาโดยสมบูรณ์

---

## 5. ฝ่ายบัญชี (Accountant)

**หน้าที่:** กระทบยอดธุรกรรมทั่วไป จัดการสมุดรายวัน ตรวจสอบภาษีและปิดงวด

- **การตรวจสอบสมุดบัญชีรายวันทั่วไป (General Ledger):**
  - ตรวจสอบรายการลงบัญชีเดบิต/เครดิต ที่ถูกโพสต์โดยอัตโนมัติจาก POS, จำนำ และออมทอง แบบเรียลไทม์
- **การปิดงวดบัญชี (Period Lock):**
  - ปิดรอบเพื่อป้องกันไม่ให้พนักงานในร้านกรอกข้อมูลย้อนหลัง หรือทำการเปลี่ยนแปลงตัวเลขในอดีตเด็ดขาด โดยระบุวันที่ล็อกงวด

---

## 6. ผู้ดูแลระบบ (Admin)

**หน้าที่:** จัดการรายชื่อผู้ใช้ระบบ กำหนดสิทธิ์การเข้าถึง และการสำรองข้อมูลฐานข้อมูล

- **การจัดการผู้ใช้และบทบาท (Users & RBAC):**
  - สร้างบัญชีใหม่, กำหนดสิทธิ์บทบาทหน้าที่ (Owner, Cashier, etc.) ผูกบัญชีผู้ใช้เข้ากับสาขาที่รับผิดชอบ
- **การสำรองข้อมูลระบบ (Backup):**
  - รันสคริปต์สำรองข้อมูลตามรอบเวลา (เช่น ทุกเที่ยงคืน) และเก็บรักษาไฟล์สำรอง SQL อย่างปลอดภัยตามเอกสาร Runbook
